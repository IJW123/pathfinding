# p11 — Iterative collision solver + FixedUpdate migration

## Context

Two problems from the p10 evaluation (`plans/p10_obstacles_collision.md`):

1. **Clipping when pushing objects.** Current pipeline detects all collisions from frame-start
   positions, then resolves once with stale manifolds (`narrow_phase.rs::detect_collisions` →
   `response.rs::resolve_solid_collisions`). Multi-contact (corners) and pushable chains
   (player → quad → pentagon) over/under-correct → visible clipping.
2. **Tunneling.** Pure positional correction in `Update`: a fast step or frame hitch can cross a
   thin wall between frames.

Agreed approach: **(1)** broad phase once per tick → candidate pair list → Gauss-Seidel solver
that re-runs SAT on the pair list with current positions each iteration (slop + percentage to
kill rest jitter); **(2)** migrate movement + collision + motion measurement to `FixedUpdate`
(Bevy default 64 Hz; max step 300/64 ≈ 4.7u < 20u wall thickness ⇒ tunneling statically
impossible). Built for thousands of mostly-static objects: static/dynamic broad-phase split,
per-tick cached world shapes, AABB early-out. Push split stays 50/50 but routed through a
per-pair weight function so mass slots in later (user decision). No render interpolation
(explicit non-goal; 64 Hz stepping accepted, follow-up if visible).

On approval, save this plan as `plans/p11_collision_solver_fixedupdate.md` (project convention).

## Design decisions (validated)

- **No `RemovedComponents`** for static-index maintenance — removal events are frame-buffered;
  FixedUpdate can skip frames at high FPS and miss them. Detect removals structurally:
  rebuild when any static matches `Or<(Changed<Transform>, Changed<Collider>)>` (Changed covers
  Added) **or** static count ≠ index entry count. Tick-based change detection is robust across
  skipped fixed ticks.
- **Solver works on cached world shapes, not Transforms.** Corrections are pure translations
  (rotation never changes mid-solve), so each correction updates an accumulated `offset` and
  translates the cached `WorldShape`/`Aabb` incrementally. Transforms written back **once** at
  the end (skip `offset ≈ 0`) — one `Changed<Transform>` per moved entity.
- **Pass 0 emits events.** Pass 0 runs SAT on *all* candidate pairs (incl. non-solid), emits
  `CollisionEvent` from pre-resolution manifolds (preserves current semantics), and applies
  corrections to solid pairs. Iterations 1..N run on solid pairs only, early-exit when an
  iteration applies no correction. Static-static pairs never generated by construction.
- **Slop gates corrections, not detection.** SAT keeps `COLLISION_EPSILON`;
  `push = normal * (depth - PENETRATION_SLOP).max(0.0) * PENETRATION_PERCENT`.

## New constants (`crates/collision/src/constants.rs`)

| Constant | Value | Rationale |
|---|---|---|
| `SOLVER_ITERATIONS: usize` | 8 | 3-body chains converge in ≤4; headroom for corners |
| `PENETRATION_SLOP: f32` | 0.5 | 1.25% of player size — invisible, kills rest jitter |
| `PENETRATION_PERCENT: f32` | 0.8 | damping; residual after 8 iters ≈ depth·0.2⁸ |
| `BROAD_PHASE_MARGIN: f32` | 8.0 | > max per-tick displacement 4.7u (300 u/s ÷ 64 Hz) |

`CELL_SIZE = 80` unchanged.

## Steps

### 1. `Aabb` newtype — new `crates/collision/src/aabb.rs`
`struct Aabb { min: Vec2, max: Vec2 }` with `overlaps`, `inflated`, `translated`, `center`,
`half_extents`. Change `shape.rs::world_aabb` to return `Aabb` (replaces `(Vec2, Vec2)` tuple);
update its tests + the `spatial_hash.rs` call site (interim until step 6).

### 2. World-space SAT dispatch + translation
- `world.rs`: add `WorldShape::translate(&mut self, delta: Vec2)` (poly: shift verts; circle:
  shift center). Doc the pure-translation assumption.
- `narrow_phase.rs`: extract `test_world_pair(&WorldShape, &WorldShape) -> Option<Manifold>`
  from `test_pair`'s match body; `test_pair` = `to_world` ×2 + `test_world_pair` (existing
  test keeps passing).

### 3. Static index — new `crates/collision/src/static_index.rs`
```rust
pub struct StaticEntry { entity: Entity, shape: WorldShape, aabb: Aabb, solid: bool }
#[derive(Resource, Default)]
pub struct StaticColliderIndex { cells: HashMap<IVec2, Vec<usize>>, entries: Vec<StaticEntry> }
pub fn maintain_static_index(...)  // dirty check per design decisions; full rebuild when dirty
```
Cells from un-inflated AABB via grid math. Cached world shapes computed once (statics never move).

### 4. Broad phase — new `crates/collision/src/broad_phase.rs`
```rust
pub struct DynamicBody { entity: Entity, shape: WorldShape, aabb: Aabb, solid: bool, offset: Vec2 }
#[derive(Resource, Default)] pub struct DynamicBodies { bodies: Vec<DynamicBody> }  // allocs retained
pub enum PairBodies { DynamicDynamic { a: usize, b: usize }, DynamicStatic { body: usize, anchor: usize } }
pub struct CandidatePair { bodies: PairBodies, both_solid: bool }
#[derive(Resource, Default)] pub struct CollisionPairs { pairs: Vec<CandidatePair> }
pub fn collect_collision_pairs(...)
```
Per tick: gather dynamics (`Without<Static>`), compute `to_world` + AABB once each. For each
dynamic, inflate AABB by `BROAD_PHASE_MARGIN`:
- vs statics: iterate inflated-AABB cells in the static index, collect entry indices,
  `sort_unstable` + `dedup` (multi-cell statics), AABB-overlap test before pushing a pair.
- vs dynamics: retained per-tick grid (`Local<HashMap<IVec2, Vec<usize>>>`, cleared each run),
  index-ordered pairs + seen-set dedup (`Entity` is `Ord` — drop the `.index()` dance), AABB
  early-out.

### 5. Solver — new `crates/collision/src/solver.rs`
```rust
pub struct PairWeights { a: f32, b: f32 }
pub fn pair_weights(a_static: bool, b_static: bool) -> PairWeights  // mass slots in here later
pub struct InitialContact { a: Entity, b: Entity, manifold: Manifold }
pub fn solve_pairs(bodies, statics, pairs, contacts: &mut Vec<InitialContact>) -> SolveOutcome  // pure
pub fn resolve_collisions(...)  // system: solve_pairs → emit events → write Transforms once
```
Correction per pair (normal a→b): a gets `-push * w.a`, b gets `+push * w.b`; each application
updates `offset`, `shape.translate`, `aabb.translated`. Early exit when an iteration applies
nothing. Pure core keeps tests App-free.

### 6. Rewire plugin, delete dead code
- `plugin.rs`: init `StaticColliderIndex` + `DynamicBodies` + `CollisionPairs`; systems
  `(maintain_static_index, collect_collision_pairs, resolve_collisions).chain().in_set(CollisionSet)`
  in **FixedUpdate**.
- Delete `response.rs` (semantics absorbed; port its test into solver tests). Delete
  `detect_collisions`. Rename `spatial_hash.rs` → `grid.rs`: keep pure `cell_of` +
  `cells_for_aabb(&Aabb)`; drop `SpatialHash` resource + `rebuild_spatial_hash` (no external
  users — verified). Update `lib.rs` module list.

### 7. FixedUpdate migration
- `crates/player/src/plugin.rs`: `move_player` → `FixedUpdate`, still `.before(CollisionSet)`.
  Body unchanged (`Res<Time>` inside FixedUpdate yields fixed dt).
- `crates/motion/src/plugin.rs`: `measure_velocity` → `FixedUpdate` in `MotionSet`; update the
  scheduling-contract doc comment.
- `crates/app/src/main.rs`: `configure_sets(FixedUpdate, MotionSet.after(CollisionSet))`.
- `crates/hud/src/plugin.rs`: drop `.after(MotionSet)` + import (cross-schedule ordering is
  meaningless; HUD reads last completed tick — one frame staleness, fine). Camera untouched.

### 8. Tests
- `aabb.rs`: overlaps / inflated / translated.
- `world.rs`: translate poly + circle. `narrow_phase.rs`: `test_world_pair` convention.
- `static_index.rs` (App-level): populated on first run; no rebuild when clean; rebuild on
  Transform change; entry gone after despawn (count-mismatch path).
- `broad_phase.rs`: multi-cell entity pairs once; near-contact within margin enters list;
  beyond margin doesn't; static-static absent.
- `solver.rs` (pure, App-free): corner two-wall resolve (residual ≤ slop on both, no
  push-through); chain A→B→C-static; separated pair → pass 0 only, zero offsets; overlap < slop
  → no correction (rest jitter); symmetric 50/50 split; pass-0 contacts include non-solid pairs
  with a→b normal + pre-resolution depth; port `dynamic_a_pushed_opposite_normal`.

### 9. Housekeeping
`./bin/housekeeping.sh` (clippy + fmt) clean; `cargo test -p collision -p motion -p player`.

## Verification (end-to-end)
`cargo run -p pathfinding`: push the quad into the pentagon and into a wall corner — no clipping,
chain pushes propagate within a tick, objects settle without jitter. Hold a direction into a wall
corner — player slides, no penetration, no oscillation. HUD speed still reads sensibly.

## Risks / edge cases
- Dynamic pushed across a cell boundary mid-solve: pair list frozen post-broad-phase; margin 8.0
  bounds it; pathological pileups self-correct next tick (residual > slop → re-paired).
- Messages across fixed ticks: 0 ticks/frame can double-deliver to Update readers; none exist —
  document that future sensors must read in FixedUpdate.
- Pairs first touching mid-solve emit no event this tick (will next tick, residual ≥ slop).
- Resting contacts emit an event every tick — matches current semantics, intentional.
- `WorldShape::translate` breaks if anything rotates colliders mid-solve — documented invariant.
